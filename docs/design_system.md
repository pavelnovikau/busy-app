# BUSY Design System
## busy.bar + busy.app — Color Scheme & Visual Language

**Дата:** 2026-04-08
**Проект:** mt-025-pn-busybar
**Назначение:** Референс для создания сайта-презентации

---

## Обзор: два визуальных языка одной экосистемы

Бренд BUSY использует **два разных визуальных регистра** для двух продуктов:

| | busy.bar (hardware) | busy.app (software) |
|---|---|---|
| **Настроение** | Премиум. Чистота. Apple-like | Интенсивность. Фокус. Тёмный режим |
| **Фон** | Белый / светло-серый | Чёрный |
| **Акцентный цвет** | Оранжевый | Красный |
| **Типографика** | Pragmatica Next VF (= "Source" в Tilda) | Pragmatica Next VF + Inter / Inter Display |
| **Ощущение** | Аппаратный продукт на полке | Приложение в состоянии фокуса |

## Типографика

> **Ключевая находка:** оба сайта используют один и тот же шрифт — **Pragmatica Next VF** (variable font от ParaType). На busy.bar он задекларирован как `font-family: "Source"` из-за особенностей Tilda CMS. На busy.app назван правильно — `Pragmatica Next VF Variable`. Файл один: `pragmatica-next_vf.woff2`.
>
> Платформы: **busy.bar** = Tilda, **busy.app** = Framer

### busy.bar

**Шрифт:** `Pragmatica Next VF` (variable font) — задекларирован как `"Source"` в Tilda  
URL файла: `tildacdn.one/.../pragmatica-next_vf.woff2`  
Fallback: `Arial`

| Элемент | Размер | Weight | Использование |
|---------|--------|--------|---------------|
| H1 hero | 62px | 450 | Главные заголовки секций |
| H2 | 42–62px | 460 | Крупные секционные заголовки |
| H2 subtitle | 20px | 480 | Подзаголовки секций |
| H3 body | 18–20px | 460–480 | Описания фич |
| Navigation | 18px | 480 | Пункты меню |
| Монospace | JetBrains Mono | — | Технические детали, код |

**Особенности:** Очень лёгкий weight (450–480 против стандартного 400/700). Создаёт ощущение элегантности. Межстрочный интервал стандартный (1.2).

### busy.app

**Шрифты (из @font-face CSS):**
1. `Pragmatica Next VF Variable` — заголовки, акцентный текст (тот же файл, что на busy.bar)
2. `Inter` (400, 500, 700 normal + italic) — основной body-шрифт  
3. `Inter Display` (400, 500, 700 normal + italic) — display-заголовки  
4. `Inter Variable` — вариативная версия Inter

Fallback: `Arial` (через Placeholder @font-face с `ascent-override`)

| Элемент | Размер | Weight | Использование |
|---------|--------|--------|---------------|
| H1 | 62px | 500 | Главный hero-заголовок |
| H2 | 43px | 400 | Секционные заголовки |
| H3 | 28px | 500 | Фича-заголовки в карточках |
| Navigation | 22px | 400 | Пункты меню |
| Body | 16–18px | 400 | Описательный текст |
| Монospace | JetBrains Mono | — | Таймер, числа |

**Особенности:** Использует Inter Display для feature-карточек (чуть более дисплейный вариант). Линия на 62px создаёт мощный контраст с мелким текстом.

## Компоненты и паттерны

### busy.bar

**CTA-кнопки:**
- Фон: `#EA5212` (оранжевый)
- Текст: белый, капслок (`PRE-ORDER`)
- Border-radius: ~12px (скруглённый прямоугольник)
- Размер: крупные, prominently placed

**Карточки фич:**
- Фон: белый или `#F5F5F7`
- Иконки: линейные, минималистичные (`stroke`)
- Подписи: тёмный текст + серый вспомогательный

**Навигация:**
- Фон: белый с логотипом (BUSY Bar + выпадающий)
- Правый угол: badge "Works with BUSY App"
- Активный пункт: чёрный, остальные серые

**Тёмные секции** (developer/specs):
- Фон `#050506`, текст белый
- Визуально выделяют технический контент

**Product explainer sections:**
- Один крупный hardware-объект в центре композиции
- Вокруг объекта тонкие annotation/callout-линии
- Подписи выстроены как инженерные объяснения, а не как маркетинговый grid
- Secondary copy светло-серый, чтобы не спорить с устройством
- Акцентный оранжевый используется точечно: CTA, аппаратные детали, selected state

**Card language:**
- Карточки белые или почти белые, с очень мягкой тенью
- Скругления крупные, но не “bubble UI”
- Иконки линейные или очень лёгкие монохромные
- Контент внутри карточек строится сверху вниз: headline → short body → compact bullet list

**Line / annotation language:**
- Callout-линии тонкие, спокойные, инженерные
- Линии не должны доминировать над устройством; это направляющие, а не графика ради графики
- Если секция сложная, сначала читается объект, потом связи, потом текст

### busy.app

**Hero-компоновка:**
- Левая половина: логотип + заголовок + CTA
- Правая: mockup телефона
- Нет явной навигации-сетки — fullscreen

**Feature-карточки (Work/Rest):**
- Work: красная карточка `#570F11` с телефонным mockup
- Rest: зелёная карточка `#104619` с телефонным mockup
- Текст: белый, с иконками emoji 🖥️ / ☕

**Feature list (таймер, блокировщик):**
- Иконки красные (круглые, заполненные)
- Bold-слово + серое описание: "**Active timer.** See your time left…"
- Layout: текст слева, mockup справа

**Мобильный mockup-стиль:**
- iPhone 14 Pro-подобный (Dynamic Island)
- Снимки показывают реальный UI приложения

### Strategy / Ops UI

Это третий визуальный слой экосистемы BUSY. Он не повторяет напрямую landing-page busy.bar и не должен выглядеть как экран busy.app. Это аналитический, редакторский и системный интерфейс для карт, роадмапов, диаграмм и product thinking.

**Назначение:**
- strategy map
- ecosystem view
- roadmap / feature graph
- insight board
- internal product tools

**Основной характер:**
- Светлая тема по умолчанию
- Очень высокий уровень визуального воздуха
- Тихий, almost-editorial UI
- Красный BUSY-акцент только для focus/selection/critical path
- Минимум декоративных эффектов

**Типографика:**
- `Geist` — основной UI-шрифт для strategy/ops
- `Geist Mono` — labels, ids, chips, legends, technical annotations
- Причина: в системных интерфейсах Geist читается лучше и практичнее, чем брендовый display-подход busy.bar/busy.app

**Композиционные правила:**
- Один главный объект внимания на экран: карта, диаграмма, список или панель
- Secondary UI должен уступать контенту по контрасту и плотности
- Focus mode не перестраивает смысл интерфейса, а только усиливает главный путь
- Любая сложная схема должна читаться в три шага:
  1. структура
  2. выделенный путь
  3. пояснения

**Диаграммы и карты:**
- Сначала видны ring/container shapes
- Потом node hierarchy
- Потом связи
- Потом labels
- Связи должны помогать чтению логики, а не создавать отдельный визуальный шум

**Sidebar language:**
- Плотная, но тихая
- Небольшой моноширинный сервисный текст
- Основные пункты крупнее, чем служебные
- Focus / selection state выделяется цветом и лёгкой заливкой, а не тяжёлым chrome

**Roadmap language:**
- Списки должны читаться без hover
- Focus mode в roadmap приглушает лишнее, но не ломает scanability
- Anchor items выделяются сильнее, supporting items слабее
- Неиспользуемые items dim, но остаются читаемыми

## Рекомендации для презентационного сайта

### Цветовая схема

Рекомендуем взять **hybrid approach** — объединить два регистра в один:

```css
/* Backgrounds */
--bg-primary:     #000000;    /* busy.app: тёмный основной */
--bg-surface:     #111111;    /* карточки на тёмном */
--bg-light:       #F5F5F7;    /* busy.bar: светлые секции */

/* Brand colors */
--accent-red:     #ED0018;    /* Primary CTA, акценты */
--accent-orange:  #EA5212;    /* Hardware CTA, устройство */
--accent-green:   #104619;    /* Rest/success states */

/* Text */
--text-primary:   #FFFFFF;    /* на тёмном */
--text-primary-dark: #000000; /* на светлом */
--text-muted:     #7A7A86;    /* вторичный на обоих */

/* Semantic */
--work-phase:     #570F11;    /* красноватый фон работы */
--rest-phase:     #104619;    /* зелёный фон отдыха */
```

### Типографика для сайта

```css
/* Рекомендации — повторяем стратегию оригинала */
font-family: 'Pragmatica Next VF', 'Pragmatica Next', sans-serif; /* бренд-заголовки */
font-family: 'Inter Display', 'Inter', sans-serif;                 /* секционные заголовки */
font-family: 'Inter', sans-serif;                                  /* body */
font-family: 'JetBrains Mono', monospace;                          /* числа, таймер, код */

/* Источник шрифта (самохостинг или CDN): */
/* Pragmatica Next — коммерческий, от ParaType. Лицензия нужна. */
/* Альтернатива без лицензии: Neue Haas Grotesk / Aktiv Grotesk / DM Sans */

/* Scale */
--text-hero:  62px / weight 500   /* hero заголовки */
--text-h2:    43px / weight 400   /* секционные */
--text-h3:    28px / weight 500   /* фичи */
--text-body:  18px / weight 400   /* описания */
--text-small: 15px / weight 400   /* подписи */
```

### Ключевые UI-паттерны сайта

1. **Тёмный hero** с красным акцентом на ключевое слово
2. **Work/Rest split-card** (красная + зелёная) — визуальный якорь бренда
3. **Горизонтальный timeline-слайдер** для настройки таймера
4. **Phone mockup** как главный hero-элемент справа
5. **"Features in development" секция** → Discord CTA
6. **Большие типографические заголовки** (~62px) на белой/тёмной секции

## Что добавить в систему как правило

На основе текущих экранов busy.bar система должна описывать не только токены, но и layout grammar:

1. **Hero object first** — сначала объект, потом текст, потом связи.
2. **Explainer sections** — сложные секции строятся через callouts, а не через обычный 3-column grid.
3. **Technical calm** — developer-friendly блоки выглядят инженерно и спокойно, без “dashboard noise”.
4. **Air as hierarchy** — иерархия создаётся не только цветом и размером, но и количеством пустого пространства.
5. **Accent discipline** — оранжевый и красный не размазываются по интерфейсу; акцент ставится в 1-2 местах на экран.
6. **Strategy UI exception** — для внутренних карт и роадмапов допустимо использовать `Geist`, сохраняя BUSY palette и compositional restraint.
