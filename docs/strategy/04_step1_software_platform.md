# Шаг 1: Software Platform Strategy
**Временной горизонт:** 0–6 месяцев после launch  
**Фокус:** Программная ценность при существующем железе  
**Дата:** 2026-04-08

---

## TL;DR

> **Задача Ring 1:** Превратить BUSY Bar из "дорогого таймера со светом" в незаменимый центр управления вниманием. Без нового железа. Только через программное обеспечение.

**Одна строка:** *Пока ты работаешь — BUSY App следит за всеми твоими статусами автоматически.*

> **Важно:** Ring 1 = `Auto Presence`. Analytics-heavy слой вынесен в отдельную подфазу Ring 1.5 (`Focus Memory`), чтобы не перегружать первый release.

---

## Для кого

**Основной пользователь:** Focus Worker (разработчик, дизайнер, менеджер продукта, писатель)
- Работает удалённо или в гибридном формате
- Использует минимум 2 из: Google Calendar, Slack, Zoom/Teams, ноутбук
- Устал обновлять статусы вручную
- Знает, что прерывания убивают продуктивность, но не может их контролировать

**Вторичная аудитория:** Семья и ближайшие коллеги — они видят результат, но не устанавливают приложение

## Что строим: 4 ключевые возможности Ring 1

### Возможность 1 — Auto-Status Engine
*"Приложение само знает, когда ты занят"*

**Как работает:**
1. BUSY App подключён к Google Calendar / Outlook
2. За 2 минуты до встречи → BUSY Bar автоматически меняется на 🔴 (занят)
3. Zoom/Teams/Meet открылся → переключается на 🔴 + включает "Recording" анимацию
4. Встреча закончилась → 15 минут "остывания" → переключается на 🟢 (доступен)
5. Параллельно: обновляется Slack status + Teams presence

**Почему это важно:**
- Убирает главное трение: помнить обновить статус перед каждым звонком
- Работает для семьи даже если они не знают что такое Slack
- Одно действие (синхронизировать календарь) = вся цифровая жизнь синхронизирована

**Экраны:**
- Setup wizard: подключение календаря за 3 клика
- Status timeline: что произойдёт сегодня по расписанию
- Override button: "сейчас я занят" даже без встречи в календаре

---

### Возможность 2 — Focus Analytics Dashboard
*"Узнай, как на самом деле проходит твой день"*

**Что отслеживается автоматически:**
- Deep Work Time: когда BUSY Bar был в режиме "занят" (красный/оранжевый)
- Meeting Time: автоопределение через Zoom/Calendar
- Available Time: когда был в зелёном статусе
- Interrupted Sessions: как часто прерывался до истечения таймера

**Ключевые виджеты:**
- **Daily Focus Map** — тепловая карта: каждые 30 минут окрашены по типу активности
- **Interruption Cost** — сколько времени потеряно на context switches. Stanford: 1 прерывание = 23 мин восстановления. BUSY Bar знает каждый прерванный таймер
- **Focus Score** (0–100) — агрегированная метрика дня
- **Best Time to Work** — паттерн твоих лучших часов за последние 30 дней
- **Weekly Streak** — количество дней с ≥4 часами фокуса подряд

**Почему это важно:**
- Нет другого инструмента, который даёт эти данные автоматически (не через ручной input)
- BUSY Bar — единственное устройство, которое physically знает, когда ты работаешь
- Данные = аргумент перед руководителем: "вот мои 6 часов deep work в день"

> **Scope note:** в рамках Ring 1 мы проектируем эту модель данных и базовые счётчики, но полноценный analytics-first слой, app tracking, auto-timesheet и interruption cost переходят в Ring 1.5.

---

### Возможность 3 — Shared Presence (Family + Team)
*"Кто угодно может видеть твой статус без установки приложения"*

**Family Mode:**
- Генерируется короткий URL: `busy.app/status/pavel`
- Страница показывает: 🔴 Занят до 17:00 / 🟢 Свободен
- Опция: кнопка "Мне нужно поговорить" → тихое уведомление на телефон
- Работает на любом телефоне/планшете без регистрации

**Team Presence (опциональный):**
- Видеть статусы коллег в BUSY App (только с их разрешения)
- Aggregate view для руководителя: кто в deep work прямо сейчас
- Privacy-first: каждый сам решает, кто видит его статус

**Почему это важно:**
- Семья получает ценность без BUSY Bar и без регистрации
- Создаёт новый social contract: "красный = не входить" без слов
- Viral loop: семья видит → рассказывает другим → новые продажи

---

### Возможность 4 — Desktop App (macOS + Windows)
*"BUSY Bar работает в фоне, ты не думаешь о нём"*

**Что делает:**
- Menubar / System tray иконка — текущий статус всегда виден
- Автозапуск при старте компьютера
- Горячие клавиши: `⌥⌘B` → быстрое переключение статуса
- Native notifications: "Через 5 минут встреча, BUSY Bar переключится"
- Background sync: не нужно держать браузер открытым

**Почему это важно:**
- Web app = забываешь закрыть вкладку → disconnect
- Desktop app = устройство всегда онлайн, всегда синхронизировано
- Горячие клавиши = меньше friction, чем тянуться к устройству

## Пользовательский путь (User Journey)

### День 1: Setup (< 15 минут)
```
Коробка открыта
    → BUSY Bar включен, подключен к WiFi
    → QR-код на экране → скачать BUSY App
    → Подключить Google Calendar (2 клика)
    → Выбрать: какой цвет = занят, какой = свободен
    → Готово. BUSY Bar знает твоё расписание.
```

### День 3: Открытие ценности
```
Утро: BUSY App показывает план дня с автостатусами
    → 9:30 встреча → BUSY Bar сам переключился
    → Zoom открылся → переключился
    → Коллега написал в Slack "ты занят, ладно потом"
    → [AHA MOMENT] — "оно работает без моего участия"
```

### День 7: Привычка
```
Analytics появляются (нужна неделя данных)
    → "У тебя 3.2 часа фокуса вчера. Обычно у тебя лучшие сессии 10:00-12:00"
    → Пользователь настраивает "Protected Hours" 10:00-12:00
    → BUSY Bar автоматически в DND в эти часы
```

### День 30: Evangelism
```
Коллега спрашивает: "Что это у тебя на столе?"
    → "Это показывает всем что я занят"
    → "И само переключается?"
    → [VIRAL MOMENT]
```

## Метрики успеха

| Метрика | Baseline | Target (6 мес) | Критический порог |
|---------|----------|----------------|-------------------|
| **30-day retention** | — | ≥ 40% | < 20% = остановить всё |
| **Calendar connected** | 0% | ≥ 60% пользователей | < 30% = rethink onboarding |
| **Daily Active Use** | — | ≥ 35% DAU/MAU | < 15% = product problem |
| **Shared URLs created** | 0 | ≥ 40% пользователей | — |
| **App Store Rating** | — | ≥ 4.5 ⭐ | < 4.0 = quality crisis |
| **Support tickets** | — | < 5% пользователей/мес | > 15% = UX problem |

## Что мы НЕ делаем на этом шаге

| Идея | Почему не сейчас |
|------|-----------------|
| SDK / API для разработчиков | SDK уже существует (Python/TS/Go, анонсировано Day 1). Ring 2 = Developer Portal + App Library + community вокруг готового SDK |
| Matter / Home Assistant | Ring 2 — зависит от стабильного WiFi layer |
| Team enterprise dashboard | Ring 2 — нужны данные о product-market fit для B2C сначала |
| BUSY Bar Mini | Ring 3 — не каннибализируем до валидации |
| AI recommendations | Ring 3 — нужно накопить данные |
| Full app tracking / timesheet / interruption economics | Ring 1.5 — второй слой после Auto Presence |

---

## Риски и митигация

| Риск | Вероятность | Действие |
|------|------------|---------|
| Apple/Google ограничат доступ к Calendar API | Средняя | Использовать официальные OAuth, избегать scraping |
| Zoom/Teams меняют detection API | Высокая | Fallback на process detection (имя приложения в taskbar) |
| Privacy-сопротивление (tracking рабочего времени) | Средняя | Privacy-first messaging, local storage, no cloud required |
| Desktop app сложнее поддерживать чем web | Высокая | Electron/Tauri с shared codebase, prioritize stability over features |

## Connection к Ring 2

Успешное выполнение Ring 1 создаёт фундамент для платформы:
- Стабильная база пользователей с **накопленными данными** → AI in Ring 3
- Доказанный **product-market fit** → Developer SDK имеет смысл
- Shared URL механизм → **Team Mesh** расширяется естественно
- Desktop app → **API bridge** для developer интеграций

## Connection к Ring 1.5

После успешного Ring 1 мы не прыгаем сразу в platform narrative.  
Следующий опциональный слой — `Focus Memory`:

- app tracking только во время BUSY-сессий
- auto-timesheet
- interruption cost
- best hours / score

Это отдельная подфаза, потому что она усиливает продукт, но не нужна для первого aha moment.
