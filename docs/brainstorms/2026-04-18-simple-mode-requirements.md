# Simple Mode — Упрощённая версия для команды и партнёров

## Problem Frame

Текущая full-версия приложения перегружена для первого знакомства. Реальная обратная связь: человек открыл страницу, увидел кольца и не понял ни что это, ни зачем. Нет нарратива — есть только структура.

Simple mode решает это: рассказывает историю BUSY Bar через формат Desire → Problem → Solution → Transformation. Аудитория — команда и партнёры, затем потенциальные пользователи. Цель первых 3 секунд — зацепить эмоцию, не объяснять структуру.

## Requirements

- R1. На корневом маршруте `/` показывается landing screen с двумя кнопками: **SIMPLE** и **FULL**. Одна строка-удар под кнопками (или над). FULL ведёт на текущую стратегическую страницу (`/strategy`).

- R2. SIMPLE (`/simple`) — вертикальная последовательность из 5 полноэкранных (100dvh) секций. Каждая секция = одна мысль: заголовок + одно предложение поддержки. Скролл только вертикальный.

- R3. Нарратив строго следует формату **Desire → Problem → Solution → Transformation**:

  | # | Глава | Заголовок | Поддержка |
  |---|-------|-----------|-----------|
  | 1 | Desire | Я хочу заканчивать день с ощущением, что сделал что-то важное. | У каждого есть это желание. Мало у кого — инструмент для него. |
  | 2 | Problem | 1 прерывание = 23 минуты. Каждый раз. | Flow, Toggl, Qbserve — всё есть по отдельности. Единого потока нет. |
  | 3 | Solution — Сигнал | BUSY Bar: физический якорь. | Красный свет на столе. Сначала вручную — потом система сама знает. _(Но твои инструменты всё ещё не знают друг о друге.)_ |
  | 4 | Solution — Платформа | Всё твоё окружение узнаёт, когда ты занят. | Slack, Calendar, Smart Home синхронизируются. _(Но кто помогает тебе стать лучше?)_ |
  | 5 | Transformation | Система наблюдает. Ты растёшь. | AI Coach превращает твои данные в инсайты. Суверенитет над вниманием — это не функция. Это цель. |

- R4. Нарратив использует паттерн «Yes, but...» для R0→R1→R2: каждая фаза закрывает проблему предыдущей и открывает следующую — создаёт нарастание к трансформации.

- R5. Каждая секция имеет акцентный цвет из дизайн-системы: Desire=`--tx` (нейтральный), Problem=`--re` (red), Solution-Сигнал=`--re`, Solution-Платформа=`--or` (orange), Transformation=`--bl` (blue). Цвет применяется к заголовку или маркеру секции.

- R6. Внизу экрана — sticky bottom bar высотой `--nav-height: 56px`. В баре: иконка «оглавление» слева + название текущей главы по центру. Тап открывает bottom sheet с 5 главами-ссылками и кнопкой **«Полная версия →»** внизу шторки.

- R7. Bottom sheet (оглавление) показывает семантические имена глав: «Желание», «Проблема», «Сигнал», «Платформа», «Трансформация». Каждая — ссылка-прокрутка к секции.

- R8. Кнопка **«Полная версия →»** в конце bottom sheet и на последней секции ведёт на `/` (текущий маршрут full-версии, который переместится на `/strategy`).

- R9. Дизайн строго из дизайн-системы: только CSS-переменные из `tokens.css` (цвета, шрифты, spacing, радиусы). Никаких hex-значений и magic numbers в компонентах.

- R10. Шрифты: заголовки секций — Geist Variable, крупный кегль (`--text-3xl` / `--text-4xl`). Поддерживающий текст — `--text-base` / `--text-lg`, цвет `--tx-2`. Pixel-шрифт опционально для акцентных элементов (счётчик "23 мин").

- R11. Mobile-first. Минимальная ширина 320px. На десктопе секции центрируются, max-width ~560px, padding по бокам.

- R12. Анимации — только через `motion` (уже в проекте). Fade-in при появлении секции в viewport (intersection observer). Никаких параллакс-эффектов — чистота важнее зрелищности.

## Success Criteria

- Человек, незнакомый с продуктом, за 2 минуты понимает: что это, какую проблему решает, как будет развиваться.
- После прочтения simple-режима хочется нажать «Полная версия».
- Дизайн неотличим по качеству от остального приложения (те же токены, те же компоненты).

## Scope Boundaries

- Simple mode — только чтение, нет интерактивных диаграмм, нет SVG колец.
- Нет отдельного роутера для simple / full — landing на `/`, simple на `/simple`, full переезжает на `/strategy`.
- Нет CMS — контент хранится в JSON (`src/data/simple-story.json`), не хардкожится в компонентах.
- Нет анимаций переходов между секциями при скролле (кроме fade-in) — простота первична.
- Нет A/B тестирования, feature flags.

## Key Decisions

- **Storytelling format Desire→Problem→Solution→Transformation**: подтверждён пользователем. Это не вариант — это требование.
- **Паттерн "Yes, but..."** в Solution-секциях: каждая фаза раскрывает следующий слой проблемы — иначе теряется нарастание.
- **Имена глав семантические** (не R0/R1/R2/R3): для TOC используем «Желание», «Проблема», «Сигнал», «Платформа», «Трансформация» — понятно без технического контекста.
- **Маршрутизация**: `/` становится landing с выбором, `/simple` — новый маршрут, `/strategy` — текущая full-версия (переименование `/`).
- **Данные в JSON**: `src/data/simple-story.json` — единственный источник контента для simple mode.

## Dependencies / Assumptions

- React Router v7 уже есть — добавляем маршруты `/` (landing), `/simple`, переименовываем текущий `/` в `/strategy`.
- `motion` уже в проекте — используем для fade-in анимаций.
- Bottom sheet — новый компонент, либо base-ui Popup/Drawer (уже есть в стеке).
